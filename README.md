# Discord Music Bot

Музыкальный бот для Discord с управлением из Telegram.

Возможности:

* Яндекс Музыка (поиск по названию и по ссылке)
* **Интерактивный выбор**: при поиске по названию выбирай из топ-5 результатов (кнопки)
* **Пульт управления** у сообщения "сейчас играет" (skip, pause, resume, volume)
* **Импорт плейлистов** Яндекса по ссылке целиком
* Голосовые каналы Discord
* Очередь воспроизведения с автопереходом
* Пользовательские плейлисты
* Регулировка громкости (по серверам)
* Режимы повтора
* Красивые Embed-сообщения
* Управление из Telegram: кнопочная панель и команды (несколько пользователей)
* Авто-кик за запретные слова в чате

# Установка и запуск

1. Установить зависимости:

   ```bash
   pip install -r requirements.txt
   ```

   `PyNaCl` обязателен для голоса. `aiogram` — для Telegram. `vosk`/`numpy`/`discord-ext-voice-recv` нужны только для голосового `/listen`.

2. Создать `.env` по образцу `.env.example` (см. таблицу переменных ниже).

3. Положить `ffmpeg.exe` по пути из `FFMPEG_PATH`.

4. Запустить:

   ```bash
   python bot.py
   ```

   Discord и Telegram-бот поднимаются вместе, в одном процессе.

# Переменные .env

```text
DISCORD_TOKEN=...        # токен Discord-бота (обязательно)
YANDEX_TOKEN=...         # токен Яндекс Музыки (обязательно)
FFMPEG_PATH=.\ffmpeg\bin\ffmpeg.exe   # путь к ffmpeg (без кавычек и без r)
GUILD_ID=...             # ID сервера — для мгновенной синхронизации слэш-команд

TELEGRAM_TOKEN=...       # токен Telegram-бота от @BotFather (опционально)
TELEGRAM_ADMIN_ID=...    # твой Telegram ID; несколько — через запятую
VOICE_CHANNEL_ID=...     # ID голосового канала для музыки из Telegram

VOSK_MODEL_PATH=model    # путь к модели Vosk (только для /listen)
```

* Свой Telegram ID можно узнать у бота `@userinfobot`.
* `GUILD_ID` и `VOICE_CHANNEL_ID` копируются правым кликом по серверу/каналу при включённом Режиме разработчика в Discord.

# Discord-команды

## Голосовой канал

* `/join` — подключиться к голосовому каналу.
* `/leave` — отключиться.

## Воспроизведение

* `/play <название|ссылка>` — найти трек и добавить в очередь. Если ищешь по названию — показывает 5 результатов с кнопками выбора. Если ссылка на трек — добавляет сразу. Если ничего не играет — запускается сразу; иначе встаёт в конец. Бот сам заходит в канал.
* `/import <ссылка>` — загрузить весь плейлист/альбом Яндекса по ссылке (например, `https://music.yandex.ru/playlist/123456`).
* `/skip` — пропустить текущий трек.
* `/pause` — пауза.
* `/resume` — продолжить.
* `/stop` — остановить и очистить очередь.
* `/volume <0-200>` — громкость (сохраняется для сервера).
* `/nowplaying` — текущий трек с пультом управления (кнопки skip, pause, resume, volume).

## Очередь

* `/queue` — текущий трек и очередь.
* `/clear` — очистить очередь.
* `/shuffle` — перемешать.
* `/remove <номер>` — удалить трек из очереди.
* `/move <откуда> <куда>` — переставить трек в очереди.
* `/loop off|track|queue` — режим повтора.

## Плейлисты

* `/playlists` — список своих плейлистов.
* `/playlist_create <название>` — создать.
* `/playlist_show <название>` — показать содержимое.
* `/playlist_delete <название>` — удалить.
* `/playlist_add <название> <трек|ссылка>` — добавить трек.
* `/playlist_remove <название> <номер>` — удалить трек по номеру.
* `/playlist_play <название>` — добавить плейлист в очередь и запустить.
* `/playlist_rename <старое> <новое>` — переименовать.

# Авто-кик за слова

Кто напишет в чат «банворд» или «банворд» (в любом регистре) — бот кикает автора.

Требования:

* В Developer Portal включён **Message Content Intent**.
* У роли бота есть право **Kick Members**, и роль бота стоит **выше** кикаемого (владельца сервера кикнуть нельзя).

Список слов — в `KICK_TRIGGERS` в начале `bot.py`.

# Управление из Telegram

Бот поднимает Telegram-управление, если задан `TELEGRAM_TOKEN`. Доступ — только у пользователей из `TELEGRAM_ADMIN_ID` (несколько через запятую), остальных бот молча игнорирует. Это касается и команд, и нажатий кнопок.

Музыка из Telegram играет в канал `VOICE_CHANNEL_ID`.

## Кнопочная панель (основной способ)

`/start` или `/menu` — открывает пульт с кнопками. Сверху видно: что играет, сколько в очереди, громкость и режим повтора. После каждого нажатия панель обновляется.

Кнопки: ⏸ Пауза, ▶ Продолжить, ⏭ Пропустить, ⏹ Стоп, 🔀 Перемешать, 🧹 Очистить, 🔉 −10 / 🔊 +10 (громкость шагом 10%), 🔁 Повтор (по кругу off→track→queue), 📜 Очередь, 📁 Плейлисты, 🔄 Обновить.

**Добавить трек** — просто пришли его название или ссылку обычным сообщением (без команды). Бот добавит трек в очередь и покажет панель.

**Плейлисты** — кнопка «📁 Плейлисты» показывает список плейлистов кнопками; тап проигрывает весь плейлист. «⬅ Назад» — в меню.

## Команды (тоже работают)

Музыка: `/play <название|ссылка>` (если название — выбирай из 5 результатов), `/search <название>` (выбор из топ-5), `/import <ссылка>` (загрузить плейлист), `/skip`, `/pause`, `/resume`, `/stop`, `/volume <0-200>`, `/queue`, `/np`, `/clear`, `/shuffle`, `/remove <номер>`, `/move <откуда> <куда>`, `/loop off|track|queue`, `/join`, `/leave`.

Плейлисты: `/playlists`, `/playlist_create <название>`, `/playlist_show <название>`, `/playlist_delete <название>`, `/playlist_add <название> | <трек|ссылка>`, `/playlist_remove <название> <номер>`, `/playlist_play <название>`, `/playlist_rename <старое> | <новое>`.

`/help` — текстовая справка по командам.

Особенности формата (Telegram передаёт аргументы одной строкой):

* `/playlist_add` и `/playlist_rename` используют разделитель `|`.
* `/playlist_remove` берёт номер последним словом, остальное — название.

> Плейлисты в Telegram у каждого пользователя свои (ключ `tg:<id>`) и отдельны от плейлистов в Discord — чтобы при нескольких людях списки не смешивались. Очередь и плеер при этом общие на сервере.

# Технические заметки

* Discord и Telegram-бот работают в одном процессе на общем asyncio-цикле: `asyncio.gather(client.start(...), dp.start_polling(...))`. Telegram-хендлеры дёргают те же функции плеера, что и слэш-команды.
* Прямые ссылки Яндекса протухают, поэтому в очереди и плейлистах хранится `track_id` + `query`, а ссылка на поток получается заново перед воспроизведением.
* Блокирующие запросы к Яндексу выполняются в отдельном потоке (`asyncio.to_thread`), чтобы бот не подвисал.
* Автопереход на следующий трек — через `after`-колбэк FFmpeg с пробросом корутины в event loop.
* Токены берутся только из `.env`; файл в `.gitignore`.
* Слэш-команды синхронизируются в `GUILD_ID` (мгновенно). Без него — глобально, с задержкой до часа.
* Требуется Python 3.13.4+ — в более ранних 3.13 при старте возможно зависание на медленном WMI-вызове внутри `platform` (баг самого Python).
